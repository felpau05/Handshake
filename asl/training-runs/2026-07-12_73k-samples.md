Sources: a Kaggle ASL alphabet image set (auto-labeled via MediaPipe) merged
with live webcam collections from multiple players, so the model isn't
purely trained on one person's hand geometry.

## Method

- **Input:** 21 MediaPipe hand landmarks (x, y, z) → normalized to be
  translation-, scale-, and mirror-invariant (wrist-relative, scaled by
  wrist→middle-knuckle distance, left hand mirrored to right) → 63-dim vector.
- **Model:** small MLP, 64 → 32 → 28 units (ReLU, 20% dropout, softmax output).
- **Split:** 85% train / 15% validation, random per-sample.
- **Training:** 60 epochs, batch size 256, Adam (lr=0.001), categorical
  cross-entropy. Ran on the native `@tensorflow/tfjs-node` backend
  (multi-threaded CPU, AVX2/FMA) — **~5 minutes** for the full 73k-sample run.

## Caveat worth stating out loud in a presentation

This validation split is **random within the same collection sessions**, not
held out by person — so 99.3% measures how well the model fits data it's
already seen *similar* hands for, not true generalization to a brand-new
person's hands. In practice this showed up as real players getting
noticeably fewer letter commits than the players whose hands are
overrepresented in the training set. The fix in progress is simply more
players contributing their own collected samples.
